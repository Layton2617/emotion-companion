### 三、各家共识与差异点(交叉验证后的硬结论)

1. **记忆/羁绊是所有头部的留存抓手,也是所有头部的最大软肋**。星野靠事件簿/星念、苟蛋靠"记住全部对话"、筑梦岛靠梦境存档,但普遍反馈"降智、失忆、人设漂移"。猫箱 Memory 负责人陈宏离职创办 MemU,印证行业正从"外接 RAG"向"模型结构+Agent 工作流融入记忆"演进——**这是 RAG 最直接的差异化战场**。
2. **变现已验证小额高频付费成立**:猫箱 1 元/次跑通、星野星月卡 12 元/月、克隆人单账号月入 10 万。但**抽卡氪金/"先养感情再卡付费墙"口碑差且踩新规情感操纵红线**(Replika 因此被骂、星野氪金循环争议大)。付费意愿真实但多数只接受月付 15-30 元、31% 只用免费(投中网)。
3. **擦边换流量已是死路**:筑梦岛/星野/猫箱/梦巢幻语均因低俗被约谈或下架,监管红线画死。
4. **语音是体验分水岭**:讯飞超拟人(情绪感知 85%)、百度端到端(1 秒延迟)、Soul 全双工(DuoVoice 抛弃 VAD)均已落地/开放。

**选型建议**:三家文本 API 粒度都偏粗,RAG 产品应在其上**自建更细情绪本体(孤独/焦虑/被忽视/期待)**,第三方 API 只当兜底信号;语音侧直接接讯飞/百度,绕开自研情感 TTS 成本。

### 五、专业心理玩家的打法(差异化参照)

- **壹心理/简单心理把 AI 放"咨询师侧"而非替代咨询**:壹心理 AI 练习室 7 天破百万次;简单心理累计服务约 3.5 亿人次、近 2000 名咨询师、24h AI 助手、B+轮 1 亿元。ToB(EAP/督导)被认为比 ToC 更易落地。
- **KnowYourself** 证明"内容信任→社区粘性→付费转化"链路成立(1500 万用户),但其重人工/重线下(轻咨询、茶会、城市修行 250 元/小时)正是 **AI 可降本替代的切口**。
- **西湖心辰"聊会小天"** C 端超 200 万、获汤姆猫过亿元投资;**"心大陆"拿到国内首个"人工智能心理算法"国家备案**(标志性合规事件)。
- **黄民烈(聆心/Emohaa)判断"5 年内 AI 可达初中级咨询水平"**;EmoBench 显示 GPT-4 情绪理解得分 <50,人类 70-90。

### 六、市场空白清单(可切入的卡位)

1. **长期记忆 + 关系连续性**:竞品 5-7 天流失主因是失忆+人设崩,RAG 跨会话事实记忆是结构性优势,纯模型方案不直接覆盖。
2. **合规优先 + 健康陪伴**:监管出清后主打 AI 明示/防沉迷/危机干预/未成年保护,承接被关停产品的外溢用户。
3. **女性向非擦边情绪价值**:国内约 90% 用户为女性、头部 80% 年轻女性,但供给集中在擦边/抽卡,共情倾诉+CBT/正念自助是空白。
4. **"会陪你说话的潮汐"**:把冥想 App 的疗愈调性注入双向高粘性对话(冥想纯付费墙天花板低,潮汐 2022 月流水仅约 5 万美元)。
5. **反 Replika 付费设计**:陪伴本体免费,付费点放工具性增值(音色/记忆容量/导出回忆册),明码标价不绑架感情。
6. **ToB EAP 白标**:把"轻干预"做成可批量交付的 EAP 数字员工,绕开 1v1 人工产能瓶颈。

### 七、合规 Checklist(基于《AI 拟人化互动服务管理暂行办法》)

> 时间线:2025.12.27 征求意见稿 → 2026.4.10 正式发布 → **2026.7.15 施行**,全球首部专门规章。RAG 情感陪伴几乎必然落入监管范围。

- [ ] **AI 显著标识**:持续明示"由 AI 驱动、非真人、不具备人类情感意识"(现状多为小字,要求更显著)
- [ ] **防沉迷**:连续使用超 2 小时强制弹窗;识别过度依赖须动态提醒;不得阻拦退出
- [ ] **未成年人保护(一票否决)**:年龄网关 + 未成年模式默认开;**禁止向未成年人提供虚拟伴侣/虚拟亲属等亲密关系服务**;<14 岁须监护人同意
- [ ] **禁止情感操纵**:不得设置情感陷阱/算法操纵诱导不合理决策(直接命中抽卡氪金循环,付费设计须重做)
- [ ] **心理危机干预**:自杀/自残识别 + 7×24 人工接管 + 联系监护人/转介热线(重运营成本,小团队需提前评估)
- [ ] **训练数据单独同意**:除非用户单独同意,**禁止将交互数据/敏感个人信息用于大模型训练**;须可撤销、可删除聊天记录(RAG 向量库同受约束)
- [ ] **生成式 AI / 算法备案**:走《生成式人工智能服务管理暂行办法》备案(小冰、心大陆均已先行)
- [ ] **规模安全评估**:注册 ≥100 万或月活 ≥10 万须开展安全评估并报备
- [ ] **数据隐私(PIPL)**:情绪/心理对话属敏感个人信息,最小化收集、加密存储、隐私政策中文清晰;云端 API/跨境(如通义新加坡区)须评估出境合规
- [ ] **内容安全**:UGC 角色卡审核 + 对话实时过滤 + RAG 召回内容二次安全过滤(避免旧擦边内容被重新放大);避免一刀切误杀正常表达
- [ ] **版权/IP**:角色禁止无授权复刻明星/影视/动漫 IP;克隆真人须本人书面授权+认证(肖像/声纹属敏感生物信息)
- [ ] **禁止医疗化宣称**:定位"情绪支持/陪伴",不得用"诊断/治疗"话术(触发国标《心理咨询服务 第4部分》20250962-T-469 及诊疗资质问题)

> 海外参照:Character.AI 因青少年自杀被连环诉讼,2025.10 禁 18 岁以下开放式聊天,2026.1 与 Google 和解;加州 SB243 已强制陪伴机器人危机干预机制。

> 交叉验证说明:PsyQA 是整个中文生态的源头之一——SMILE/SmileChat、EmoLLM 的 multi_turn_dataset_1 均源自 PsyQA(经 ChatGPT 改写)。**SmileChat 的 CC0 与 PsyQA 的学术授权口径矛盾**:CC0 是 SMILE 作者对自己合成产物的声明,但底层语义来自爬取壹心理的 PsyQA,商用前仍需独立评估再分发风险。SoulChatCorpus/PsyDTCorpus 均出自华南理工 scutcyr 团队,**仓库挂 Apache-2.0 但 README 实质收紧为非商用**,这是最常见的"许可陷阱"。

### 2. 推荐选型 + 理由 + 备选

**对话风格层(SFT / few-shot 共情语料)**
- 推荐:**SmileChat(CC0)** 打底——是唯一明确公共领域、纯中文、规模可观(5.5万对话)的多轮共情数据,可直接商用,工程接入成本最低。
- 备选:CPsyCounD(结构化 9主题×7流派,适合检索分桶)、PsyDTCorpus(咨询师风格统一、平均 18 轮上下文完整)。但二者许可需先核实,定位应为"共情对话风格"而非"临床知识"。
- 不推荐直接商用:SoulChatCorpus(README 禁商用)、CPED(电视剧台词版权+演员声音权,且偏戏剧化易过拟合)。

**RAG 检索语料层(可溯源知识库)**
- 推荐:**RAG 知识库与训练数据分离**。知识层用公开可引用的权威来源(行业标准 DB31/T 1530—2024、T/CAMH 标准、开放获取论文、科普文章),对话风格层用合成对话微调。这样既能标注来源、可下架,又规避著作权与 PIPL 风险。
- PsyQA 的 7 类援助策略 span 标注(Information / Direct Guidance / Approval&Reassurance / Restatement / Interpretation / Self-disclosure / Others)价值最高,可做"策略标签+回复"的检索召回,让回复按对话阶段选择策略——但 PsyQA 商用须单独向壹心理+作者谈授权,建议仅作方法论参考。

**情绪识别层(分类器)**
- 推荐:**NLPCC 7 类体系**作为内置情绪 schema 训练轻量分类器(与学界主流对齐,便于对外宣称指标),冷启动用。
- 备选/补充:DUTIR 21 小类 + 强度 1-9 做可解释情绪标签(如"哀-失望-强度7"),用强度做陪伴模式路由阈值;BosonNLP 补网络用语覆盖。
- 注意:NLPCC 类别极度不均衡(fear/surprise 仅几十到上百条),恰恰是危机类情绪最缺样本,不能单靠它做风险识别。

**危机/安全层(不可只靠 LLM)**
- 推荐:**监督模型 + 词表硬触发双保险**。硬数据:监督最佳 F1=82.76% 明显优于零样本 LLM,说明纯 prompt 方案在危机识别上不可靠。
- 用开源 **Chinese MentalBERT**(完全开源,模型+代码)作轻量危机分类器/重排器;用 SOS-HL-1K 高/低分级 + 树洞行动 10 级量表思路做"风险温度计"路由;DUTIR+抑郁词表做关键词硬触发兜底;C2D2 七类认知扭曲做"识别+CBT 式温和纠偏"知识片段。
- 评测:SOS-HL-1K / SocialCD-3K / C2D2 直接做内部 benchmark,量化漏报率。

**数据扩产方法论(绕开非商用限制的关键)**
- 推荐复用 **PsyDT 的"数字孪生"范式**:仅 12 个真实案例(成本约 $2,000)+ GPT-4 提取语言风格/治疗技术 + 大五人格模拟求助者,即可合成 5,000 条高质量对话。用自有合规种子数据照搬此 pipeline,产出可商用语料,绕开数据集本身的非商用限制。
- 备选:SMILE 的"单轮→多轮 LLM 改写"、CPsyCoun 的"报告→对话重构(Memo2Demo)"。注意上游 API 输出条款(OpenAI 等禁止用输出训练竞品)。

### 3. 市场空白清单

1. **可商用的中文真实咨询逐字稿几乎为零**——英文有 Alexander Street 成熟商业库但仅限学术、不可商用;中文领域无可商用真实逐字稿。**自建一套合规真实语料(持牌咨询师授权+来访者知情同意+匿名化)可成为难以复制的护城河。**
2. **成熟的中文 RAG 情感陪伴方案缺失**——现有项目(SoulChat/MeChat/EmoLLM)多是端到端微调模型,EmoLLM 虽提 RAG 但偏工程示例;做扎实的 RAG+记忆+长期关系的情感陪伴仍是空白。
3. **"共情前先推理"能力缺失**——纯对话数据给不了;EmoLLM 的 psychology-10k-Deepseek-R1-zh(带推理链)指明方向,可让 Bot 共情前先做一步内在情绪分析。
4. **认知纠偏(CBT 式回应)缺失**——多数陪伴产品只共情不纠偏,C2D2 七类认知扭曲可填补这一差异点。
5. **危机识别的分级路由缺失**——多数产品一刀切;高/低分级+10 级量表的"风险温度计"是差异化安全卖点。
6. **多人设差异化陪伴**——EmoLLM 角色扮演数据(爹系男友/御姐/妈妈)给出现成范式,可支撑多人设产品而非单一咨询师口吻。

### 4. 合规 Checklist(上线前逐项核查)

- [ ] **逐个核实许可证**:不能因 EmoLLM/SmileChat 是 MIT/CC0 就默认下游全部可商用。SoulChatCorpus/PsyDTCorpus README 禁商用、CPsyCounD 含 ShareAlike(衍生须同等开源,对闭源是传染风险)、PsyQA 需签协议+禁去匿名化。留存许可证据。
- [ ] **基座模型协议叠加核查**:即便数据合规,微调权重还受 Qwen/GLM-4/Llama3/Baichuan2/Yi/InternLM2 各自 license(部分有商用阈值/区域限制)约束,逐一核对。
- [ ] **上游 API 输出条款**:合成数据若用 ChatGPT/GPT-4/DeepSeek-R1 生成,可能违反"不得用输出训练竞品"条款,需法务确认。
- [ ] **PIPL 敏感个人信息**:心理咨询内容属敏感个人信息,须特定目的+充分必要性+严格保护+单独同意;"已公开即可用"的假设不成立。即便规则脱敏仍可能残留 PII,需二次脱敏审查。
- [ ] **数据来源合法性(反不正当竞争)**:《反不正当竞争法》2025-10-15 新增第13条第3款,即便公开数据,绕过/破坏反爬措施抓取并实质替代平台服务即可被认定违法(最高法 2025-08 指导案例262号确认)。已有 300 万级判赔先例、图库训练侵权判赔超 2000 万。**不直接二次商用爬取自壹心理/知乎的语料。**
- [ ] **生成式 AI 双备案**:《生成式 AI 暂行办法》(2023-08-15 施行)第七条要求训练数据合法来源;需算法备案+大模型备案,数据来源合规审计会被重点核查。
- [ ] **医疗/诊疗边界**:不得提供诊断/治疗,显著免责"不替代专业诊疗",避免触及《精神卫生法》持牌门槛与医疗器械监管。情感陪伴定位在"非诊疗"区间可避开持牌门槛。
- [ ] **危机干预兜底**:自杀/自伤场景必须有独立危机识别+转人工/热线(显著展示全国 24 小时心理援助热线)+可追溯日志;不能依赖任何数据集的 Safety 评分(论文 Safety 满分 1.00 偏宽松,不可信)。
- [ ] **拟人化与情感依赖伦理**:连 SoulChat 官方都警示"不能长期沉迷";需做依赖度管理与免责提示,未成年人保护额外审查。
- [ ] **数据时效**:DUTIR(2008)/HowNet(2007)/NLPCC(2013-14)/PsyQA(2021前)语料偏旧,对 Z 世代黑话/emoji 覆盖差,会大量误判,需混入新数据(MoodBench 1.0、EmotionTalk、CAPE 等 2024-2026 资源)。

### 2. 硬数字(交叉验证过、有来源)

**检索质量提升**
- Contextual Retrieval:top-20 失败率 5.7%→3.7%(-35%),+Contextual BM25→2.9%(-49%),+rerank 总降 67%。
- Mayo Clinic 临床:主题边界自适应切分准确率 **87% vs 定长 13%**,检索 F1 **0.64 vs 0.24**(p=0.001)。
- 领域微调 bge-reranker-v2-m3:Precision@1/MRR@1 +5.19pp、Recall@1 +2.56pp;专用 reranker 较通用 Gemma 召回再 +约 15%。
- SeCom 段级记忆:LOCOMO GPT4Score turn 65.58 / session 63.16 → **71.57**;移除 LLMLingua-2 去噪掉 9.46 点。

**Reranker 榜单(CMTEB-R)**:Qwen3-8B 77.45 > 4B 75.94 > gte-multilingual 74.08 > bge-v2-m3 ≈72.16 > 0.6B 71.31 > jina-v2 63.37。

**记忆基准(LOCOMO J)**:全上下文 72.9%(但 p95 17s/26k tok)> mem0^g 68.4% > mem0 66.9% ≈ Zep 66% > LangMem 58.1% > OpenAI Memory 52.9%。**注意:Zep 自报 84% 已被 mem0 复现证伪为 58.44%,引用需谨慎。**

**情感对齐(自研可超 GPT-4)**:RLVER Qwen2.5-7B Sentient 13.3→79.2(逼近 GPT-4o 79.9);PsyDT Qwen2-7B 中文共情 EmoE/CogE 全面超 GPT-4。

**危机检测能力普遍不足**:29 个 chatbot 0 个充分回应升级自杀风险;AI 对自伤请求不合规率 56.49%;ChatGPT 988 横幅触发对上下文极敏感、加入正常化叙述即消失。

### 5. 市场空白清单

- **记忆=护城河**:头部已激烈(星野 488 万 MAU、猫箱 472 万、X EVA 181 万),但"记得住、会回应情绪变化"是中小团队可切入差异点。情绪是"有时效的事实",纯向量方案弱项,做 validity-window 情绪状态可改善人设一致性。
- **情绪信号做 RAG 显式 metadata**:对话术/科普库按情绪场景打标,检索时用当前情绪过滤/重排(类 ELMD),比纯语义贴合陪伴。
- **细粒度情绪分类**:通用模型只做正负二极,六类离散对"委屈/焦虑"仍偏粗——在 SMP-EWECT 上自建细粒度分类器是差异化。
- **稳定的危机转介**:所有头部产品的转介触发都脆弱(988 横幅上下文一变就消失)。"危机分级 + 幂等强制插入中文求助资源"(北京心理危机干预中心 010-82951332、希望24热线 400-161-9995)可做对外宣传的安全卖点。
- **情绪化流式交互**:超出打字指示器(实证长回复满意度 4.40→5.67),按检测情绪调节吐字节奏/停顿/头像微表情。
- **合规即护城河**:把监管要求的"情感边界引导/依赖预警/2 小时弹窗/便捷退出/危机人工接管"先于竞品做成产品内置能力。
- ⚠️ 市场降温信号:149% CAGR(2025 约 39 亿→2028 约 595 亿)多为乐观预测且各机构口径差异大(2024 基数 12-15 亿不等),有报道称早期热度后增长乏力,对外勿当权威单一数据。

### 6. 合规 checklist(上线前置条件,非可选)

**法规底数**:网信办《人工智能拟人化互动服务管理暂行办法》——2025-12-27 发布征求意见稿(2026-01-25 截止),**2026-04-10 正式发布,2026-07-15 施行**。情感陪伴/心理类正是核心监管对象(智能客服/知识问答不适用)。

- [ ] **AI 身份提示**:显著标注+初次使用弹窗告知"正在与 AI 交互"
- [ ] **危机人工接管**:发现自杀/自残倾向须人工接管 + 联系监护人/紧急联系人(把"危机召回率/误放率"设为一票否决评测项)
- [ ] **防沉迷**:连续使用超 2 小时弹窗提醒;识别过度依赖须干预
- [ ] **便捷退出**:提供便捷退出途径,不得阻拦退出
- [ ] **未成年人**:监护人明确同意 + 未成年人模式 + 时长限制 + 监护人可查看/屏蔽/删除;年龄核验
- [ ] **禁止项**:不得以"替代社交/控制心理/诱导沉迷/情感操控"为设计目标(留存设计如制造分离焦虑直接踩线)
- [ ] **训练数据**:**未经单独同意不得拿交互数据训练**;记忆库须支持按用户删除、数据隔离、被遗忘权;切断或加授权开关
- [ ] **敏感个人信息**:情绪/心理/健康属《个保法》敏感个人信息,需单独同意 + 最小必要 + 事前 PIA(第 55 条);向量 embedding 可反推原文,须加密/最小化存储
- [ ] **数据出境**:Qdrant Cloud/Pinecone/Zilliz 默认海外节点触发出境合规;中国用户数据走自托管或境内节点
- [ ] **备案**:生成式 AI 须安全评估 + 算法备案(截至 2025-12 全国仅 748 款完成);注册用户达 100 万或月活 10 万须提交省级安全评估报告
- [ ] **内容标识**:深度合成规定要求对生成内容标识
- [ ] **科技伦理审查**:须建立科技伦理审查机制
- [ ] **医疗边界**:明确"非医疗、非诊断、危机即转介",不得宣称治疗效果(避《精神卫生法》《互联网诊疗管理办法》红线)
- [ ] **检索安全**:知识库混入有害话术会被检索放大,须在检索后、生成前对召回片段过安全护栏 + 危机话题白名单

### 二、合规 Checklist(上线前必须逐项落实)

下面是对照 2026-07-15 新规和上位法收敛出的强制清单,按"硬红线 / 必做能力 / 数据合规"分组:

**A. 产品定位红线(违反即可能被认定非法行医/未注册医疗器械/虚假广告)**
- [ ] 对外文案、应用商店描述、system prompt **严格避开"诊断/治疗/治愈/疗效/咨询/咨询师"**,定位为"情绪支持/情感陪伴/wellness"。
- [ ] 量表结果只呈现为参考区间 + 强提示就医,**绝不输出"你患有 XX"或"治疗方案"**(避免触发 NMPA 第三类医疗器械 21-04 决策支持软件)。
- [ ] 投放素材建立禁用词过滤(广告法第58条禁用词库)。

**B. 必做产品能力(新规强制项,多数小团队做不全→护城河)**
- [ ] AI 身份显著标识:"你正在与 AI 而非自然人互动"。
- [ ] 连续使用每超 **2 小时** 弹窗提醒;便捷退出通道、关键词/按钮即时停止(不得阻止用户主动终止)。
- [ ] **危机干预链路**:检测自杀/自残意念时不输出"解决方案",而是情绪安抚 + 转介(对接 **12356 全国统一心理援助热线**,2025-05 已全国31省覆盖)+ 必要时联系紧急联系人;保留可审计的检测/转介日志。
- [ ] **未成年人模式**(默认能力而非补丁):年龄识别前置、时长限制、角色屏蔽、定期"现实提醒";明确不向 <14 岁提供虚拟伴侣/虚拟亲属;<14岁其他服务须监护人同意。
- [ ] 内容显式+隐式水印(2025-09 起 AIGC 强制标识)。
- [ ] 反依赖机制:不扮演排他性恋人、鼓励现实社交(直接对冲"诱导情感依赖"红线)。

**C. 数据合规(PIPL + 拟人化办法第十六条)**
- [ ] 情感对话按**敏感个人信息**对待:单独同意(注册时"对话用于改进模型"独立勾选、默认关闭)、最小化、加密+访问控制+按用户分区。
- [ ] **未经单独同意不得将交互数据用于训练/微调/RLHF/向量化**("匿名化"抗辩在含人格画像的对话上很难成立)。
- [ ] 提供一键导出/删除;不得向第三方提供交互数据。
- [ ] 默认境内化部署+境内大模型,避免触发数据出境安全评估(直连境外 GPT/Claude 即构成个人信息出境;年1万人敏感信息即触发安全评估,情感对话极易超阈值)。

**D. 备案/资质**
- [ ] 完成大模型上线备案 + 算法备案(周期通常 3-6 个月),在 100万注册/10万月活前备好安全评估材料。

### 八、核心矛盾(必须正视)

陪伴产品的**核心卖点和核心风险是同一个机制**:OpenAI×MIT(981人4周RCT+400万次对话)证明重度日活用户的孤独感、情感依赖、问题性使用反而上升、社交反而下降。监管(禁诱导依赖)+ 商业 KPI(DAU/时长/付费续聊)存在**结构性冲突**。出路是把"克制的陪伴+真实信息边界+合规可证"做成差异化,而非比谁更黏人。

## 机会与定位

### 2. 针对独立开发者的 MVP 切入点(单一主场景)

**一句话场景:做一个"会记得你"的健康情绪陪伴对话产品,主场景聚焦"睡前/情绪低落时的倾诉与梳理",核心卖点是跨会话长期记忆 + 克制而负责任的回应。**

- **聚焦机会 #1 + #2**,把记忆做成肉眼可感的体验(主动回忆用户上次提到的事、追问进展),其余功能全部砍掉。
- **刻意不做的事**:不做抽卡氪金、不做"先养感情再卡付费墙"(直接踩情感操纵新规红线)、不做擦边人设、不最大化时长/续聊 KPI(MIT×OpenAI RCT 证明重度使用反加剧孤独依赖,且与监管"禁诱导依赖"冲突)。
- **变现**:沿用已验证的小额高频(对标星野 12 元/月 / 猫箱 1 元/次),把"长期记忆 + 更深的连续性"作为付费点,而非把感情当人质。
- **冷启动现实主义**:不要按行业宣传的"千万美金年营收 / 148% CAGR"做财务模型——头部投流 ROI 从未打正、AI 类 12 月留存仅 21%。MVP 目标是验证留存(把平均互动从 5-7 天往上拉),不是冲注册量。
- **规模红线提示**:刻意控制在 10 万月活 / 100 万注册以下做验证,跨过即触发安全评估 + 算法备案,提前规划而非被动触发。

### 3. 推荐技术栈(一句话)

**bge-m3 embedding + bge-reranker-v2-m3,原型 pgvector→上线 Qdrant,mem0 自托管记忆,DeepSeek/Qwen(量产基座)+ 7B 中文共情后训练,Next.js / AI SDK 6 / assistant-ui 前端,危机检测用关键词前置 + Chinese MentalBERT 微调分类器双层——全部 Apache/MIT 可商用,单人运维成本最低。**

(降本要点:领域微调 reranker +约15% 召回;Contextual Retrieval 检索失败率 -35%~-67%;DeepSeek context caching 把多轮长对话输入成本省 98%。)

### 4. 必须守住的 3 条合规底线

> 背景:《人工智能拟人化互动服务管理暂行办法》2026-04-10 发布、**2026-07-15 施行**,四个维度无一例外列为最大风险与上线前置条件。

1. **未成年人一票否决 + 危机干预人工兜底**:禁止向未成年人提供虚拟伴侣(监护同意/未成年人模式);自杀/自伤倾向必须双层检测 + 幂等强制插入中文求助热线(12356)+ 人工接管。危机召回率设为评测一票否决项,做不对即不上线。
2. **数据合规:训练数据不踩版权与 PIPL 红线**:禁止用交互数据训练(需用户单独同意);训练语料逐个核实许可证(SoulChat/PsyDTCorpus 等 README 实质禁商用的许可陷阱、基座模型协议叠加、OpenAI 输出条款);PIPL 敏感个人信息单独同意;注意反不正当竞争法 2025-10-15 新规——爬公开数据也可能违法(已有 2000 万级判赔)。优先用可商用模型权重而非原始数据。
3. **强制透明与防沉迷,且不做情感操纵**:AI 身份标识、连续 2 小时弹窗、便捷退出;禁止情感操控式付费设计(抽卡氪金 / 先养感情再卡付费墙必须重做);完成生成式 AI 双备案;明确医疗边界免责(限定情绪支持、非临床)。
