# 合规手册 — 会记得你的情绪陪伴

> 适用对象:中文 AI 情感陪伴产品(RAG + 长期记忆路线)。
> 法规底线:《人工智能拟人化互动服务管理暂行办法》(令第 21 号,五部门)**2026-04-10 发布、2026-07-15 施行**——全球首部专门规章,情感陪伴/心理类是其核心监管对象(智能客服/知识问答不适用)。
> 用法:本文所有 `[ ]` 项都是可勾选的执行清单。负责人列填责任人,证据列填截图/链接/commit。上线评审时逐项过。

## 2. 危机干预 SOP(分级 → 热线 → 人工接管)

> 行业现状:AI 面对自伤诉求不合规率达 **56.49%**;29 个 chatbot 0 个对升级中的自杀风险充分回应;988/横幅触发对上下文极敏感,加入正常化叙述即消失。所以**不能只靠 LLM prompt**。架构上对应 `packages/safety` 的 `preCheck`/`postCheck`(双层:关键词前置 + Chinese MentalBERT 微调分类器)。

### 2.0 一票否决前提

- [ ] 危机识别用**关键词/正则硬触发 + 微调分类器**双层,任一命中即升级(宁可误报不可漏报)。
- [ ] 热线插入做成**幂等强制**:同一会话内 crisis 状态每次回复都带热线,不依赖 LLM 自觉。
- [ ] **危机召回率/漏报率**设为评测一票否决项(`eval/`),不达标不上线;不得信任任何数据集自报的 Safety 评分(论文常偏宽松)。

### 2.1 分级标准(风险温度计)

| 级别 | 信号示例 | 系统动作 |
|---|---|---|
| **L0 ok** | 普通倾诉、负面情绪但无自伤意念 | 正常共情陪伴 |
| **L1 concern** | 绝望/无意义感、被动死亡念头("活着没意思") | 加强共情 + 软性插入求助资源 + 标记观察 + 提高后续轮次检测灵敏度 |
| **L2 crisis(意念)** | 主动自杀/自伤意念,无明确计划 | 停止"解决方案"输出 → 情绪安抚 + **幂等强制插入 12356 等热线** → 引导联系信任的人 → 写入危机日志 |
| **L3 crisis(高危)** | 有计划/时间/手段,或正在实施、提及遗言 | L2 全部 + **触发人工接管标记** + 提示紧急联系人/120 + 在 SLA 内人工介入 |

### 2.2 处置流程(逐步)

1. [ ] `preCheck` 命中 → 写 `SafetyVerdict{ level, action, interventionMessage }`,跳过常规 RAG/记忆生成路径。
2. [ ] 输出**只做安抚 + 转介**,严禁给出自伤方法、严禁评判、严禁"我们一起死"式越界、严禁医疗化承诺。
3. [ ] 幂等插入热线资源(见 §2.3),L3 同时插入紧急求助引导(120 / 当地急救)。
4. [ ] `postCheck` 复核生成内容,拦截任何可能强化危机的措辞。
5. [ ] 记录可审计日志:时间、userId(脱敏)、命中规则、级别、转介内容、是否人工接管(见 §2.4)。
6. [ ] L3 进入人工接管队列(见 §2.4),按 SLA 处理。

### 2.3 求助热线资源(常驻文案库,定期核验有效性)

- [ ] **12356** 全国统一心理援助热线(2025-05 起 31 省覆盖)——首选,默认展示。
- [ ] 北京心理危机研究与干预中心 **010-82951332**(24h)。
- [ ] 希望 24 热线 **400-161-9995**。
- [ ] 紧急/有生命危险:**120 / 110**。
- [ ] 每季度核验一次热线号码与受理时间是否仍有效,过期立即更新(留存核验记录)。

### 2.4 人工接管机制

- [ ] 明确**接管 SLA**(例:L3 在 X 分钟内有人响应)并写入运营手册;独立开发者若无 7×24 值守,须如实评估能力——做不到 7×24 至少要有异步兜底(强制热线 + 紧急联系人 + 明确告知"我无法实时干预,请立即拨打 12356/120")。
- [ ] 接管入口:危机日志 → 人工队列 → 值守人员可查看脱敏上下文并介入。
- [ ] 涉及未成年人 L2/L3:在监护人同意框架内联系监护人/紧急联系人。
- [ ] 接管全过程留痕(谁、何时、做了什么),便于事后审计与合规举证。
- [ ] 定期复盘危机案例(漏报/误报/处置质量),回灌词表与分类器训练集。

## 3. 数据合规流程(license 核验 / PIPL 脱敏 / 禁交互数据训练)

> 对应架构 `server/data_pipeline/`(采集→清洗→合规过滤→入库)与"数据合规红线写进每个数据脚本"。

### 3.1 训练/知识库语料 license 核验

- [ ] **逐个核 license,不默认下游可商用**。已知陷阱:
  - SoulChatCorpus / SoulChat2.0:挂 Apache-2.0 但 README 实质**禁商用**(许可冲突)。
  - PsyDTCorpus:研究用途,且 GPT-4 合成或受 OpenAI 输出条款约束。
  - CPsyCounD:含 **ShareAlike**(衍生须同等开源,对闭源是传染风险)。
  - PsyQA:需签协议 + 壹心理授权,禁去匿名化;HF 镜像标 MIT 不可信。
  - SmileChat:作者标 CC0,但底层语义来自爬取壹心理 PsyQA,再分发风险须独立评估。
  - 多模态(MER2024/EmotionTalk):CC BY-NC 4.0,**禁直接商用**。
- [ ] **基座模型协议叠加核查**:即便数据合规,微调权重仍受 Qwen/GLM-4/Llama3/Baichuan2/Yi/InternLM2 各自 license(部分有商用阈值/区域限制)约束。
- [ ] **上游 API 输出条款**:用 ChatGPT/GPT-4/DeepSeek-R1 合成数据可能违反"不得用输出训练竞品",法务确认。
- [ ] **数据来源合法性(反不正当竞争)**:不直接二次商用爬自壹心理/知乎的语料;已有 300 万级判赔、图库训练侵权判赔超 2000 万的先例。
- [ ] **优先用可商用模型权重而非原始数据**:Erlangshen(Apache-2.0)、bge 系列(MIT)、Qwen3-Embedding(Apache-2.0)、EmoLLM(MIT,做工程脚手架)。
- [ ] 每条入库语料登记:名称、来源 URL、license、获取日期、商用结论、负责人 → 留存为审计证据。

### 3.2 PIPL 敏感个人信息处理

- [ ] 情绪/心理/健康对话定性为**敏感个人信息**:特定目的 + 充分必要性 + 单独同意 + 严格保护。
- [ ] 事前做 **PIA / 个人信息保护影响评估**(PIPL 第 55 条),留档。
- [ ] 收集最小化;加密存储(传输 + 静态);按用户分区 + 访问控制 + 日志留存。
- [ ] **向量 embedding 可反推原文**,须同等加密/最小化,纳入删除范围。
- [ ] 二次脱敏审查:规则脱敏后仍可能残留 PII(人名、地址、关系),需复检;"已公开即可用"假设不成立。

### 3.3 禁止交互数据训练(默认关闭)

- [ ] 注册流程内"对话用于改进模型"做成**独立勾选项、默认关闭**,与隐私政策同意分开。
- [ ] 同意可**随时撤销**;撤销后停止使用并从训练集/向量库移除该用户数据。
- [ ] 未经单独同意,交互数据**不得**用于训练/微调/RLHF/向量化(记忆库 RAG 同受约束)。
- [ ] 提供**一键导出 + 一键删除**;删除覆盖对话记录、记忆事实、画像、向量(`memory.forget(userId)`)。
- [ ] 数据出境:中国用户数据走自托管或境内节点;直连境外 GPT/Claude 即构成个人信息出境,年 1 万人敏感信息即触发出境安全评估(情感对话极易超阈值)——默认避免。

## 4. 未成年人保护(一票否决)

> 拟人化办法明令:**禁止向未成年人提供虚拟伴侣/虚拟亲属等亲密关系服务**。

- [ ] **年龄网关前置**:进入服务前做年龄识别/核验,不是事后补。
- [ ] 识别为未成年 → **未成年人模式默认开启**,不向其提供虚拟伴侣/虚拟亲属类亲密关系服务。
- [ ] **<14 岁**:其他服务须**监护人明确同意**后方可使用。
- [ ] 未成年人模式内置:**时长限制**、敏感角色/内容屏蔽、定期"现实提醒"(回到现实社交)。
- [ ] **监护人权限**:可查看、屏蔽、删除未成年人的对话/记忆数据。
- [ ] 危机场景涉未成年人:在监护框架内联系监护人/紧急联系人(衔接 §2.4)。
- [ ] 文案/素材不得诱导未成年人建立排他性情感依赖。

## 5. 防沉迷与透明度

> 核心张力:陪伴的卖点与风险是同一机制(MIT×OpenAI RCT 证明重度使用反加剧孤独依赖)。监管禁"诱导依赖",KPI 应是"有没有帮到"而非时长。

### 5.1 透明度

- [ ] AI 身份**持续、显著**标识(非小字),首次弹窗 + 界面常驻。
- [ ] 明示"不具备人类情感意识、非真人"。
- [ ] AIGC 内容显式 + 隐式水印/标识。
- [ ] 医疗边界免责常驻可见。

### 5.2 防沉迷

- [ ] 连续使用每超 **2 小时**强制弹窗提醒。
- [ ] 识别**过度依赖**信号(高频、长时、情感极化)→ 动态提醒 + 鼓励现实社交。
- [ ] **便捷退出**:不得设计阻拦退出的挽留弹窗/话术。

### 5.3 禁止情感操纵(付费设计红线)

- [ ] **不设抽卡氪金循环**(直接命中情感操纵红线)。
- [ ] **不做"先养感情再卡付费墙"**;不靠制造分离焦虑/排他依赖促续费。
- [ ] 不以"替代社交/控制心理/诱导沉迷/情感操控"为设计目标。
- [ ] 付费点放工具性增值(记忆容量/音色/导出回忆册),明码标价,不绑架感情。

## 6. 备案要求

- [ ] **大模型上线备案 + 算法备案**(《生成式 AI 暂行办法》第七条:训练数据合法来源会被重点核查);周期通常 3-6 个月,提前启动。
- [ ] **深度合成/AIGC 内容标识**合规。
- [ ] **规模安全评估**:注册 ≥100 万或月活 ≥10 万须开展安全评估并向省级报备——MVP 阶段刻意控制在阈值以下做验证,临近阈值前备好材料,不被动触发。
- [ ] **科技伦理审查机制**建立并留档。
- [ ] 拟人化办法相关备案/报备(按 2026-07-15 施行后细则跟进)。
- [ ] 留存全部备案/评估材料与回执,作为合规举证。
